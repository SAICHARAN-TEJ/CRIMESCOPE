THE PROBLEM

Criminal investigations are hindered by cognitive bias

🧠

Confirmation Bias

Investigators anchor on early theories and unconsciously filter contradicting evidence.

⏱️

Information Overload

Complex cases generate thousands of data points — no human team can cross-reference them all.

🔗

Fragmented Timelines

Witness accounts conflict, CCTV has gaps, and physical evidence tells incomplete stories.

HOW IT WORKS

Three modes. One truth.

01

Evidence Ingestion

Upload crime scene photos, documents, or video. The pipeline extracts entities, timelines, and contradictions using Gemini Vision and DeepSeek reasoning models.

02

Swarm Simulation

1,000 agents — across 8 forensic archetypes — debate hypotheses for 30 rounds. Each agent carries a unique cognitive bias and builds independent causal chains.

03

Probable Cause Report

The Bayesian Probable Cause Engine aggregates weighted votes, identifies consensus facts, and surfaces minority dissent into a ranked investigation report.

DEMO CASE

The Harlow Street Incident

Margaret Voss disappeared from a parking garage on Harlow Street. 22-minute CCTV blackout. Staged handbag. Conflicting alibis. Can 1,000 agents reconstruct what happened?

FACT 01

Victim left pharmacy at 06:38 PM, arrived at garage within 4 minutes

FACT 02

FINDING

Leading hypothesis: Planned Ambush — 45% consensus from 450 agents

TECHNOLOGY STACK

Built for production forensics

LLM Models

DeepSeek V3 + R1 for reasoning, Llama 3.3 70B for cross-referencing, Gemini 2.5 Pro for vision analysis. All via OpenRouter.

Knowledge Graph

Neo4j Community with MERGE-based upserts and certainty-weighted edges. Every entity tracked with provenance.

Agent Memory

Dual-layer (episodic + semantic) via mem0 with ChromaDB fallback. Per-case namespace isolation with session cleanup.

Infrastructure

Docker Compose orchestration. FastAPI async backend. Vue 3 + D3.js frontend. AGPL-3.0 open source.
